Methods

Scope

This report includes every work in OpenAlex whose author list contains at least one affiliation to the University of Munster, identified by its stable Research Organization Registry (ROR) identifier, https://ror.org/00pd74e08. The university changed its English name from “Westfalische Wilhelms-Universitat Munster (WWU)” to “University of Munster”; the ROR record covers both names, so the crawl is not affected by the rename.

How the cost of each article is estimated

The same priority order used by the single-reference-list calculator is applied to every work:

  • If OpenAlex records an actual paid APC amount (sourced from OpenAPC), that figure is used and trusted as-is.
  • Works hosted on a recognized preprint server (arXiv, bioRxiv, medRxiv, SSRN, OSF, Zenodo, and similar) are treated as free (€0) and open.
  • Works OpenAlex classifies as diamond open access (no reader fee and no author fee) are treated as free.
  • Otherwise, for gold or hybrid open access articles without a known paid amount, the journal's list-price APC is used as an estimate. This is a list price, not necessarily what was actually paid (institutional deals, waivers, and discounts are not reflected).
  • Green, bronze, closed, and unknown access articles are recorded as having no direct APC (they may still carry a subscription cost to the university's library, which is not captured per-article by this analysis).

Open access type and free-to-read availability

The open access type is OpenAlex's own oa_status classification. Because this report covers the institution's entire output rather than one person's list, calling the Unpaywall API for every single work (as the reference-list calculator does) is not practical at this scale. Instead, “free-to-read availability” here reuses OpenAlex's own is_oa flag as a proxy, which is a reasonable but slightly less precise substitute for a direct PDF check.

Citations, journal metadata, publisher, and discipline

Citation counts, the hosting journal, the publisher, and the primary discipline are all taken directly from OpenAlex's work and source records at the time of the crawl and are not independently re-verified. Retraction status uses OpenAlex's is_retracted flag.

Refresh schedule and the crawl-in-progress indicator

OpenAlex's free tier meters usage at roughly $1 of API credit per day per contact address, resetting at midnight UTC. A university-wide crawl can involve retrieving well over one hundred thousand works, which a single day's free budget cannot complete in one pass. A scheduled GitHub Actions workflow therefore runs this crawl once a day, and the crawler checkpoints its position after every page so each run picks up exactly where the previous one stopped, whether that was because the day's free budget ran out or because the run reached its own conservative per-run limit. If a run fails outright (a bug, a network error), whatever it fetched before failing is still saved and committed, nothing is lost. The progress bar at the top of this page reflects all this: while the initial backlog is still being fetched, the figures on this page are based on a partial (but steadily growing) sample of the university's total output, not the final total. Once the backlog is complete, daily runs switch to fetching only works OpenAlex has newly indexed or updated since the previous run.

Known limitations

  • Coverage depends entirely on OpenAlex's own indexing and affiliation tagging, which is more complete and accurate for recent publications than for older ones.
  • Author-institution matching relies on OpenAlex's disambiguation; a small number of works may be mis-attributed or missed.
  • List-price APC estimates for gold/hybrid articles without a recorded paid amount may not reflect actual institutional agreements, waivers, or discounts.
  • The “first author is Munster-affiliated” filter uses OpenAlex's recorded author order and affiliation, which is not always identical to true first/corresponding authorship.
